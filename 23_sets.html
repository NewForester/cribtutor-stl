STL Associative Containers

The set and multiset Containers

set<> and multiset<> are associative containers in which value-objects are stored as keys in ascending order.

multiset<> allows duplicate keys, set<> does not. Keys are immutable.

Search and (in general) insertion operations have logarithmic complexity. Erase by iterator has constant complexity.

Iterators are bi-directional: random access is not supported. Insert and erase operations do not invalidate iterators.

Iterators are, in effect, always const since they point to immutable keys.

To use set<> and multiset<> containers include the <set> header file.

For set<> and multiset<> , the template definitions are: 

    template< typename Key, class Compare=less<Key>, class Allocator=allocator<Key> > class set;

    template< typename Key, class Compare=less<Key>, class Allocator=allocator<Key> > class multiset;

The typename Key is the type of both the key and the value-objects stored in the container.

The class Compare defines the comparator used to order objects within the container. It is an advanced option. User-defined classes must implement the pertinent operator (the default is operator<).

The class Allocator defines the allocator used when copying objects into the container. It is a very advanced option. The default uses global new() and delete().
