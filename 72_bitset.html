Other STL Containers

Bitset

A bitset<> class template is a packed, fixed size, sequence of bits. It is not an associative or even a standard container.

To use the bitset<> class template, include the <bitset> header file.

The bitset class template has a single parameter, 'N', the number of bits in the set. There are no iterators.

There are two external representations:

  1. integer is an unsigned long integer taken to be a bit-mask; the right most bit of the mask is bit 0
  2. string is a string of '0' and '1' characters; the right most character of the string is bit 0

The default constructor resets all bits. There are two additional constructors:

  1. bitset(value) the bitset is initialised from the external integer representation 'value'.
  2. bitset(str,pos=0,len=string::npos) the bitset is initialised from the external string representation 'str.substr(pos,len)'.
Any other bits are reset.

The member functions

  1. to_ulong() returns the integer
  2. to_string() returns the string representation
of the current value of the bitset.

The following member functions concern the whole bitset:

  1. any() return true if any bit is set
  2. count() return the number of bits set
  3. flip() inverts all bits
  4. none() returns true if no bit is set
  5. reset() resets all bits
  6. set() sets all bits
  7. size() returns N

The following member functions concern a single bit:

  1. flip(nn) invert bit 'nn'
  2. reset(nn) reset bit 'nn'
  3. set(nn,val=true) set bit 'nn' to 'val'
  4. test(nn) returns bit 'nn'
They throw an out_of_range exception if 'nn' is invalid.

The random access operator is defined:

  1. operator[](nn) as a rvalue, returns bit 'nn'
  2. operator[](nn) as a lvalue, sets bit 'nn'
Behaviour is undefined if ''nn'' is out of range.

Note: 'bitset[nn].flip()' does work.

The

  • bitwise,
  • shift and
  • equality
operators are all defined.

The insertion and extraction operators are also defined and convert to and from the external string representation.
