STL Associative Containers

The map and multimap Containers

map<> and multimap<> are associative containers in which '[key,value]' pair<> structures are stored in ascending order by key.

multimap<> allows duplicate keys, map<> does not. Keys are immutable.

Search and (in general) insertion operations have logarithmic complexity. Erase by iterator has constant complexity.

Iterators are bi-directional but map<> supports random access. Insert and erase operations do not invalidate iterators.

Iterators point to '[key,value]' pair<> structures so compatibility within iterators for other types of containers is limited. This affects constructors and insert operations that takes an iterator range.

To use map<> and multimap<> containers include the <map> header file.

For map<> and multimap<>, the template definitions are: 

    template< typename Key, typename T, class Compare=less<Key>, class Allocator=allocator<pair<const Key, T> > > class map;

    template< typename Key, typename T, class Compare=less<Key>, class Allocator=allocator<pair<const Key, T> > > class multimap;

The typename Key is the type of the key and typename T the type of the value-objects stored in the container. They need not be the same type.

The class Compare defines the comparator used to order objects within the container. It is an advanced option. User-defined classes must implement the pertinent operator (the default is operator<).

The class Allocator defines the allocator used when copying objects into the container. It is a very advanced option. The default uses global new() and delete().

Note, from the declaration of the allocator, what is stored in a map container is a pair<>.

There is a useful value_type defined: 

    typedef pair<const Key, T> value_type;
The first is the immutable key and the second is the value-object.
