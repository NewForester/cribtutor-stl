STL Streams

STL streams provide C++ with the input/output operations to/from

  • console,
  • files and
  • strings.

Introduction

The STL provides input and output via streams. A stream is a channel through which data is transmitted.

One end is attached to the program which can read and/or write to the stream and the other end to a device, a file or a string buffer.

For an input device such as a keyboard, the stream is attached to its source. For an output device such as a display, the stream is bound to its destination.

Typically, at the lowest level, a stream deals with characters but at the highest level it deals with C++ values. It converts between them.

There are three standard streams available to all programs: cin, cout and cerr.

There is also clog, which is a buffered version of cerr. There are wide character versions of all four.

Presumably, stream i/o could be used with any character or block device but how a stream is associated with an arbitrary device is an advanced topic.

The STL input/output library

The STL input/output library has a class hierarchy split over a number of header files.

For i/o streams there are three header files:

  1. <istream> – for input streams
  2. <ostream> – for output streams
  3. <iostream> – for input and output streams
The last effectively combines the first two.

For file and string streams there are two header files:

  1. <fstream> – for file streams
  2. <sstream> – for string streams

There are two other header files of interest:

  1. <ios> – for stream flags and such
  2. <iomanip> – for stream manipulators

There are three classes for i/o streams:

  1. istream – for input (provides extraction operators)
  2. ostream – for output (provides insertion operators)
  3. iostream – combines istream and ostream

There are three classes for file streams that inherit from the i/o stream classes:

  1. ifstream – for input from files
  2. ofstream – for output to files
  3. fstream – for file input and output

There are three classes for string streams that inherit from the i/o stream classes:

  1. istringstream – for extraction from strings
  2. ostringstream – for insertion into strings
  3. stringstream – effectively combines the other two

All methods and operators are in the std namespace.

Underneath are the classes ios and ios_base that provide definitions for stream and formatting state flags, file open modes and other constants. Many of their members (such as ios::failure and ios_base::failure) are interchangeable.

The provision of these definitions and the methods that rely on them at the lowest level means everything is available when you need it but also provides scope for some confusion. It is possible to open an ifstream for output although this serves no purpose. The seekg() methods is available for ostream objects but, again, is of no practical use.
