STL Sequence Containers

Sequence Container Methods

Constructors

The sequence containers support the standard and unremarkable:

  • default constructor,
  • copy constructor,
  • destructor and
  • operator=.

There are three additional constructors for sequence containers.

  1. The form 'sequence(nn,v_obj)' initialises a container with 'nn' copies of 'v_obj'.
  2. The form 'sequence(nn)' initialises a container with 'nn' objects created using their default constructor.
  3. The form 'sequence(first,last)' initialises a container with copies of the objects in the range '[first,last)'.

The iterator range has to refer to objects of compatible (base) type but not necessarily to a container of the same type. Anything that provides forward iterator behaviour can be used, including pointers to a C++ arrays. 

    int             array[] = {5, 4, 3, 2, 1};
    vector<int>     myVector(array, array + 5);

Iterator Methods

Each container type has four member functions that return an iterator.

  1. begin() returns an iterator that points to the first object in a container and is typically used to initialise an iterator.
  2. rbegin() returns an iterator that points to the last object in a container and is typically used to initialise a reverse iterator.
  3. end() returns an iterator that points to an imaginary object one beyond the last in the container and is typically used to determine when to stop 'iterating'.
  4. rend() returns an iterator that points to an imaginary object one before the first in the container and is typically used to determine when to stop 'reverse iterating'.

Capacity Methods

Each container type has the same capcity methods.

  1. size() returns the number of objects currently in the container.
  2. max_size() returns the implementation dependent maximum number of objects that may be stored in containers of this type.
  3. empty() returns true if the container is currently empty. It it to be preferred to 'size() == 0' for performance reasons.
  4. resize(new_size,v_obj) resizes by erasing or inserting objects at the 'end' of the container. If objects are inserted, they are copies of 'v_obj'.
  5. resize(new_size) resizes by erasing or inserting objects at the 'end' of the container. If objects are inserted, they are created using their default constructor.

Access Methods

There are four access methods.

  1. front() returns a reference to the first object in a container.
  2. back() returns a reference to the last object in a container.
  3. operator[] returns a reference to nth object in the container. There is no range check.
  4. at(n) returns a reference to the nth object in the container but performs a range check first and throws an out_of_range exception when this fails.

list<> does not support operator[] and the at() method.

Modifier Methods

There a dozen or so modifier method variants.

  1. assign(nn,v_obj) erases the current contents of a container and replaces them with 'nn' copies of 'v_obj'.
  2. assign(first,last) erases the current contents of a container and replaces them with copies of the objects within the range '[first,last)'.
  3. insert(d_it,v_obj) inserts 'v_obj' before the value-object pointed to by the iterator 'd_it'.
  4. insert(d_it,nn,v_obj) inserts 'nn' copies of 'v_obj' before the value-object pointed to by the iterator 'd_it'.
  5. insert(d_it,first,last) inserts copies of the objects within the range '[first,last)' before the value-object pointed to by the iterator 'd_it'.
  6. erase(d_it) erases the value-object pointed to by the iterator 'd_it'.
  7. erase(first,last) erases the objects within the range '[first,last)'.
  8. swap(another) swaps the contents of this container with 'another'.
  9. clear() erases all objects in the container.
  10. push_front() inserts an object at the 'front' of a container.
  11. pop_front() erases the object at the 'front' of a container.
  12. push_back() inserts a value-object at the 'back' of a container.
  13. pop_back() erases the value-object at the 'back' of a container.

vector<> does not support push_front() and pop_front() and list<> offers another nine modifier methods.

pop_back() erases the value-object at the 'back' of a container but does not return the object: use back() beforehand.

pop_front() erases the value-object at the 'front' of a container but does not return the object: use front() beforehand.

The global function swap() overloads to call the member function. The containers must be of the same type but not necessarily the same size. All iterators remain valid but afterwards point to different containers.

Beware the effect of these methods on iterators: assume all are invalidated for vector and deque containers but that all are still valid for list containers.
