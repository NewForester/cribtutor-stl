STL Algorithms

Merge and Set Algorithms

General Remarks

The algorithms in this section operate on two ordered ranges and, generally, yield a third.

The third, destination, range is assumed to be in a container long enough to receive the result. The results are undefined if either source ranges overlaps the destination.

For equivalent value-objects, relative order is preserved and those from the first source range appear before those from the second.

All these algorithms may take an optional comparator ('bfp') that implements an alternative to the default sort order less<>.

The algorithms are valid for all container types. The results are undefined if the source ranges are not ordered or are/were ordered using a different (optional) comparator.

Except for includes(), their return value is an iterator 'ret_value' such that the result is given by '[dest,ret_val)'

The Algorithms

  1. merge(first,last,start,end,dest[,bfp]) merges value-objects from the ranges '[first,last)' and '[start,end)' into the range '[dest,...)'.
  2. inplace_merge(first,middle,last[,bfp]) merges value-objects from two consecutive ranges, '[first,middle)' and '[middle,last)', within the same container.
  3. set_union(first,last,start,end,dest[,bfp]) generates the union of '[first,last)' and '[start,end)' in the range '[dest,...)'.
  4. set_intersection(first,last,start,end,dest[,bfp]) generates the intersection of '[first,last)' and '[start,end)' in the range '[dest,...)'.
  5. set_symmetric_difference(first,last,start,end,dest[,bfp]) generates the symmetric difference of '[first,last)' and '[start,end)' in the range '[dest,...)'.
  6. set_difference(first,last,start,end,dest[,bfp]) generates the difference between '[first,last)' and '[start,end)' in the range '[dest,...)'.
  7. includes(first,last,start,end[,bfp]) returns true if every value-object in '[start,end)' exists in '[first,last)', including duplicates.

For set_union() for value-objects that occur in both ranges, only those in first range are copied. Excess duplicates from either range are copied.

For set_intersection() value-objects that occur in both ranges are copied. Excess duplicates from neither range are copied.

For set_symmetric_difference() value-objects that occur in either range but not both ranges are copied. Excess duplicates from either range are copied.

For set_difference(), only value-objects in the first range that do not occur in second are copied. Excess duplicates are copied.
